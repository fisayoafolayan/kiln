# Blog API Example

A complete blog API generated by [kiln](https://github.com/fisayoafolayan/kiln) from a Postgres schema. Clone, run, and explore - no code to write.

## What You Get

From this schema:

```sql
CREATE TABLE users (
    id    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT NOT NULL UNIQUE,
    name  TEXT NOT NULL,
    role  TEXT NOT NULL DEFAULT 'member' CHECK (role IN ('member', 'moderator', 'admin')),
    ...
);

CREATE TABLE posts (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    title      TEXT NOT NULL,
    status     TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft', 'published', 'archived')),
    deleted_at TIMESTAMPTZ,  -- soft deletes
    ...
);
```

Kiln generates a full REST API:

```
GET    /api/v1/users              List users (paginated, filterable, sortable)
POST   /api/v1/users              Create a user (validated)
GET    /api/v1/users/{id}         Get a user
PATCH  /api/v1/users/{id}         Update a user (partial)
DELETE /api/v1/users/{id}         Delete a user

GET    /api/v1/posts              List posts (soft-deleted posts excluded)
POST   /api/v1/posts              Create a post
GET    /api/v1/posts/{id}         Get a post
PATCH  /api/v1/posts/{id}         Update a post
DELETE /api/v1/posts/{id}         Soft-delete a post (sets deleted_at)

GET    /api/v1/users/{id}/posts   Posts by user (nested route from FK)

GET    /api/v1/comments           List comments
GET    /api/v1/posts/{id}/comments Comments on a post (nested route)
GET    /api/v1/users/{id}/comments Comments by a user (nested route)

GET    /api/v1/tags               List tags
...plus full CRUD for comments and tags
```

All with an OpenAPI spec at `docs/openapi.yaml`.

## Prerequisites

- Go 1.26+
- Docker
- kiln (`go install github.com/fisayoafolayan/kiln/cmd/kiln@latest`)

## Quick Start (this example)

This example includes pre-made config files so you can get running immediately:

```bash
cd examples/blog-api
cp .env.example .env
make setup
make run
```

That's it. The API is running at `http://localhost:8080`.

## Starting Your Own Project

This is what you'd do on a fresh project - no pre-made configs:

```bash
# 1. Install kiln
go install github.com/fisayoafolayan/kiln/cmd/kiln@latest

# 2. Create your project
mkdir my-api && cd my-api
go mod init my-api

# 3. Start your database
#    (set up Postgres/MySQL/SQLite however you prefer)

# 4. Run kiln init - it asks 4 questions and creates kiln.yaml
kiln init

#    Database driver (postgres/mysql/sqlite): postgres
#    Database DSN (or leave blank to use an env var):
#    Environment variable name for DSN: DATABASE_URL
#    Output directory: ./generated
#    API base path: /api/v1
#
#    ✓ Created kiln.yaml
#

# 5. Generate your API
export DATABASE_URL="postgres://user:pass@localhost:5432/mydb?sslmode=disable"
kiln generate

# 6. Install dependencies and run
go mod tidy
go run cmd/server/main.go
```

After `kiln init`, edit `kiln.yaml` to configure auth, enums, hidden fields, filtering, and other overrides. Then re-run `kiln generate`.

## Try It

```bash
# List users (seeded with sample data)
curl http://localhost:8080/api/v1/users | jq

# Create a user (role is validated against enum)
curl -X POST http://localhost:8080/api/v1/users \
  -H "Content-Type: application/json" \
  -d '{"email": "dave@example.com", "name": "Dave", "role": "member"}' | jq

# Invalid role gets rejected
curl -X POST http://localhost:8080/api/v1/users \
  -H "Content-Type: application/json" \
  -d '{"email": "eve@example.com", "name": "Eve", "role": "superadmin"}' | jq
# → {"error":"validation failed","fields":{"role":"must be one of: member moderator admin"}}

# Filter users by role
curl "http://localhost:8080/api/v1/users?role=admin" | jq

# Sort users by name (descending)
curl "http://localhost:8080/api/v1/users?sort=-name" | jq

# List published posts
curl "http://localhost:8080/api/v1/posts?status=published" | jq

# Soft-delete a post (sets deleted_at, doesn't remove it)
curl -X DELETE http://localhost:8080/api/v1/posts/{post-id}

# The post no longer appears in listings
curl http://localhost:8080/api/v1/posts | jq

# Nested route: get posts by a specific user
curl http://localhost:8080/api/v1/users/{user-id}/posts | jq

# Partial update (only send fields you want to change)
curl -X PATCH http://localhost:8080/api/v1/users/{user-id} \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice J."}' | jq
```

## Features Demonstrated

| Feature | How It Works                                                                                     |
|---------|--------------------------------------------------------------------------------------------------|
| **Enum validation** | `role` and `status` have CHECK constraints → kiln auto-detects and generates `oneof=` validation |
| **Soft deletes** | `posts.deleted_at` is a nullable timestamp → kiln auto-generates soft delete behavior            |
| **Nested routes** | Foreign keys → kiln generates `GET /users/{id}/posts` and `GET /posts/{id}/comments`             |
| **Filtering** | `?status=published`, `?role=admin`, `?created_at[gte]=2024-01-01T00:00:00Z`                      |
| **Sorting** | `?sort=created_at` (ascending), `?sort=-created_at` (descending)                                 |
| **Pagination** | `?page=2&page_size=10`                                                                           |
| **Partial updates** | PATCH sends only changed fields, no `required` validation on omitted fields                      |
| **Chi router** | `framework: chi` in kiln.yaml                                                                    |
| **Auth ready** | Set `strategy: api_key` or `strategy: jwt` in kiln.yaml to enable auth middleware               |
| **OpenAPI spec** | Always in sync at `docs/openapi.yaml`                                                            |

## Project Structure

```
blog-api/
  kiln.yaml              ← kiln config (your settings)
  schema.sql             ← database schema
  seed.sql               ← sample data
  docker-compose.yml     ← Postgres container
  .env                   ← DATABASE_URL

  models/                ← bob's query builders (internal, auto-generated)
  generated/
    models/              ← request/response structs (auto-generated)
    store/               ← database operations (auto-generated)
    store/mappers/       ← type mappers (yours to edit)
    handlers/            ← HTTP handlers (auto-generated)
    handlers/helpers.go  ← error helpers, pagination (yours to edit)
    router.go            ← route registration (auto-generated)
    auth/middleware.go   ← auth middleware (yours to edit)
  docs/
    openapi.yaml         ← OpenAPI 3.0 spec (auto-generated)
  cmd/server/
    main.go              ← server entry point (yours to edit)
```

Files marked "yours to edit" are write-once - kiln generates them once and never overwrites them.

Files marked "auto-generated" are regenerated on each `kiln generate`. If you edit one, kiln detects the change and skips it (use `--force` to override).

## Customising

**Change the schema:** Edit `schema.sql`, run `make reset` to recreate the database, then `make generate`.

**Add business logic:** Edit `generated/store/mappers/users.go` to transform data, `generated/handlers/helpers.go` to change error formats, or `cmd/server/main.go` to add middleware.

**Add auth:** Edit `generated/auth/middleware.go` - it's generated with a TODO placeholder. Add your JWT validation, API key lookup, or whatever your app needs.

**Change what's generated:** Edit `kiln.yaml` - hide fields, disable operations, restrict filtering, configure enums.

## Cleanup

```bash
make down       # stop Postgres
make reset      # stop, delete data, re-setup from scratch
```
